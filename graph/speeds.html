Blue bars are write speed. White bars are read speed.

Bottom axis is number of buckets for hashtable, and the disk size the header occupied.

Test is with 971698 txs (443973598 bytes). We expect slower performance for LevelDB BTREE at higher counts, whereas the hashtree should remain constant with an unchanging load factor (number txs / number buckets).

We expect random access for Bitcoin transactions, and the dataset is huge, so cache perfomance is not important for us.
